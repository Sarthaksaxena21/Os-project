Scheduling Algorithm: Power-Aware Priority Queue

The scheduler uses a multi-level priority queue with power awareness at each decision step:

PRIORITY_SCORE = (deadline_urgency × 0.4) + (workload_weight × 0.3) + (thermal_penalty × 0.2) + (fairness_bonus × 0.1)

• Workload Prediction: EWMA of last 8 CPU bursts → classify Light / Medium / Heavy

• Core Selection: Heavy → P-Core (Big), Light → E-Core (Little), with thermal migration override

• DVFS Step: freq = f_min + (predicted_load / 100) × (f_max – f_min), rounded to nearest OPP

• Thermal Guard: If T_core > T_warn, force migration + freq step-down before scheduling
